## 0. 三个问题的直接答案

**Q2(你说得对,FDI 是质心):** 我们补测了**双向封锁**得到完整融合窗口。
prefix 封锁 [0..d)(下降点=最晚必须融合的位置)+ **suffix 封锁 [d..N)**(上升点=前 d 层
**已经**搬进 text 流的可用信号量 = 真实起点)。结果:**起点、质心、终点三者对每个模型自洽**,
中层融合不是质心伪影:

| 模型 | 融合窗口 [suffix20%↑, prefix20%↓] | 半量点(suffix 50%) | 形状 |
|---|---|---|---|
| LLaVA-1.5-7B | **[~0, 0.38]** | 0.28 | 早开始,前 1/3 内完成;首 1-2 层必要但不充分(只放开前 0.09N 层仅恢复 <20%) |
| Qwen2.5-VL-7B | [0.32, 0.57] | 0.43 | 中段窄带 |
| NEO1.0-2B | **[0.53, 0.65]** | 0.60 | 几乎原子化的中后窄带(pre-Buffer 0.30 结束后 0.2N 才发生) |
| NEO1.0-9B | [0.38, 0.62] | 0.50 | 中段 |
| Gemma-4-12B | [0.38, 0.50] | 0.48 | 中段窄带 |
| SAIL-7B | [0.53, 0.88] | 0.59 | 晚 + 长尾 |
| OneVision-7B | [0.32, 0.61] | **0.54** | **中段**(FDI 0.52 [0.42,0.58],onset80 0.50) |
| LLaVA-NeXT-7B | [0.22, 0.50] | 0.25 | **过渡态**:仍早搬运(rise50 0.25)但必要点后移(onset80 0.03→0.28)、窗口变宽有冗余(FDI 0.40 [0.30,0.50]) |

(nc:主力 6 模型各 166-200,OneVision/NeXT 各 100;全部 stratified。)

suffix 实验同时排除了"平时早融合、封锁后被后层冗余补救掩盖"的可能:若早已融合,只放开
前几层就该有信号——Qwen 和所有 native 模型都没有(<20% 直到 ≥0.36N)。

**Q1(有 encoder 为何还中层融合):** 因为**决定融合深度的不是图像 token 何时"准备好",
而是训练把"读取电路"布线在哪里**——而 LLM 的语义整合电路天然住在中层。这不是我们的特例,
是文献共识(五篇独立工作,方法互异):Kaduri 2411.17491(knockout:LLaVA ~L4-20、
InternVL2-**76B** ~L20-40/80)、Neo 2410.07149(L15-24 最强)、Zhang 2411.18620、
Devils-in-Middle-Layers 2411.16724,以及最具干预性的 **DeepInsert 2504.19327:把视觉 token
直接从中层插入、完全跳过前层,性能不掉**——前层对融合可有可无,encoder 与否改变的是图像
token 的内容,不是 LLM 读取电路的位置。真正的离群点是 LLaVA-1.5 的"早",不是 Qwen 的"中"。

**Q3(前层在干嘛):** 三件事,都有直接测量:
1. **构建问题表征(所有模型)。** lastrow 分解实验(只切答案位→图像,保留 question token 读图):
   **LLaVA min retained=0.91、Qwen=0.89**——答案位直读图像的通路几乎不承重,可用信号
   全部走 **image → question-tokens → answer** 间接路径。所以"融合的读者"是问题表征;
   它要先在前层吸收完问题文本,中层才有成熟的查询去读图。这复现并扩展了 Zhang et al.
   (2411.18620) 的两阶段图景。
2. **图像侧编码(仅 NEO/Gemma 这类带预编码结构的 native)。** CKA 显示 NEO 的图像 token 在
   pre-Buffer(前 0.30N)内完成编码;但 suffix 曲线 **0.53 才开始升**——编码完成与融合之间
   有 0.2N 空档,**证明融合等的是文本侧成熟,不是图像侧**。SAIL/LLaVA/Qwen 前层对图像
   token 改动很小(各自原因:无预编码结构 / 已被 encoder 编码完)。
3. **对 LLaVA:早期 gist 转移。** 它的融合带 [0, 0.38] 中,首 1-2 层的 text→image 读取
   已是必要的(prefix o80=0.03),但只放开前 0.09N 层只恢复 <20% 信号——即早期转移是
   分布式过程的开头,不是"一层拷贝完"。

## 1. 对"结论不显著/无规律"的回应:规律其实非常清楚

**规律 1:融合深度是稳定的模型属性,不是题目属性。** 每个模型内部跨题型(感知/OCR/推理)
FDI 几乎不变(LLaVA 0.24-0.39 全早;Qwen 0.51-0.56 全中;native 全中)。按答对/答错/
图像依赖子集分层也不变(LLaVA 图像依赖子集 0.27,其他 0.44-0.52)。

**规律 2:除 LLaVA-1.5 外,9 个配置全部中层融合,且统计显著成立。**
bootstrap 95% CI:LLaVA FDI 0.33 [0.27,0.41] 与其余所有模型分离;Qwen 0.54 [0.48,0.58]
与 native 带 [0.42,0.62] 完全重叠。"Qwen≈native"不是噪声,是 nc=166-334 下的稳定等价。

**规律 3:换度量不换结论。** NLL 度量(无字母 argmax 偏置):LLaVA 0.30 < Qwen/Gemma 0.45
< NEO 0.56/SAIL 0.58;不截断质心、onset、suffix 起点,排序全部一致。注意力深度质心
(独立测量)与 FDI 在 LLaVA(0.36 vs 0.33)、Qwen(0.53 vs 0.54)上重合。

**真正没有规律的是我 v1 的解释**,见下。

## 2. v1 解释的修正:从"视觉特征 LLM-就绪度"改为"读取电路放置"

v1 说"融合深度由视觉特征的文本对齐度/LLM-就绪度决定"。**两个事实推翻了它:**
1. [Qwen2.5-VL 的 ViT 虽 from scratch,但训练第一阶段就是 CLIP 对比预训练](https://qwenlm.github.io/blog/qwen2.5-vl/)
   ——同样是对比文本对齐的 encoder,却中层融合。"CLIP 对齐 ⇒ 早融合"被 Qwen 直接反例。
2. 按 CKA(对 DINOv2)的"特征成熟度":Qwen 和 SAIL **进场即成熟**却中/晚融合,LLaVA 进场
   是 CLIP 特征却最早——进场成熟度与融合深度**不相关甚至反相关**。

修正后的解释(也更诚实地对应 knockout 实际测的东西):**knockout 测的是"已训练电路依赖
哪里的读取",融合深度 = 训练过程把 image→question 转移电路布线在哪。** 大规模端到端多模态
训练(Qwen 18T token / NEO / Gemma / SAIL)一致地把它放在中层——与纯文本 LLM 的语义整合
位置一致(文献规律)。LLaVA-1.5 的特殊配方(**冻结** encoder + 仅 ~0.6M SFT + 弱 Vicuna)
把读取电路布线在浅层。

**已否决的备择假说**(每个都有数据):
- ~~题目难度驱动~~:类内 FDI 不变(规律 1)。
- ~~token 数/聚合瓶颈~~:Qwen 模型内 image token 比 LLaVA 的 576 还少的样本(中位 198)
  照样中层(FDI 0.47);token 数变 8 倍只移 FDI ~0.09。
- ~~度量伪影~~:规律 3。
- ~~"encoder 对齐性"~~:本节开头。
- 配方 vs LLM 强弱:由桥接模型判别(§3)。

## 3. 桥接实验:LLaVA 的"早"来自什么

LLaVA-1.5 与 Qwen2.5-VL 之间至少 4 个混淆变量(encoder 类型 / LLM 强弱 / 训练数据规模 /
token 数)。桥接模型把它们拆开:

| 模型 | encoder | LLM | 配方/数据 | 预测含义 |
|---|---|---|---|---|
| LLaVA-1.5-7B | CLIP(冻结) | Vicuna-7B | ~0.6M SFT | 基线:早 (0.33) |
| LLaVA-NeXT-7B | 同 CLIP | 同 Vicuna | anyres 多 token + 更好数据 | 若变中:token/数据;若仍早:Vicuna/配方 |
| OneVision-7B | SigLIP | Qwen2-7B | LLaVA 式但数据大得多 | 若变中:LLM/数据规模;若早:配方本身 |
| Qwen2.5-VL-7B | 自训 ViT(CLIP 预训练) | Qwen2.5-7B | 18T 端到端 | 中 (0.54) |

**结果(同分布对照:LLaVA/Qwen 限制到相同 i<500 子集:LLaVA FDI 0.31/onset 0.03,
Qwen 0.50/0.36):**

| 模型 | FDI [95% CI] | prefix onset80 | suffix rise50 | 判定 |
|---|---|---|---|---|
| LLaVA-1.5 | 0.33 [0.27,0.41] | 0.03 | 0.28 | 早 |
| LLaVA-NeXT | 0.40 [0.30,0.50] | **0.28** | 0.25 | 过渡 |
| OneVision | 0.52 [0.42,0.58] | **0.50** | **0.54** | 中 |
| Qwen2.5-VL | 0.54 [0.48,0.58] | 0.46 | 0.43 | 中 |

**判别结论:融合深度随训练规模/强度单调走向中层,与 encoder 类型无关。**
- 同 encoder 同 LLM、只升级数据+分辨率(NeXT):早读电路的**必要性**大幅后移
  (onset80 0.03→0.28、完成点 0.38→0.50),但早期搬运习惯仍残留(rise50 0.25)且出现冗余
  ——过渡态。
- LLaVA 式配方 + 强 LLM + 更大数据(OneVision):**完全中段**(0.52),与 Qwen(0.54)
  不可区分;SigLIP(同为对比式 encoder)没有带来早融合。
- 因此 LLaVA-1.5 的"早"既不是"有 encoder"也不是"对比对齐 encoder"的属性,而是
  **小规模冻结-encoder 配方在弱 LLM 上布线出的浅层读取电路**;一旦训练规模上去,
  读取电路收敛到中层——与 encoder-free 模型同一位置。**中层融合是大规模 VLM 训练的
  普遍吸引子**(与 §0-Q1 的文献共识一致)。

## 3.5 冻结实验:前融合层在给谁打工(因果级分工证据)

**方法**(`freeze_probe.py` 系列,代码经独立对抗审查,verdict sound):把某一路(image 或
text)的残差流在第 [1..k) 层冻结在其 L0 值(每层输出改写回输入;**保留第 0 层**——所有模型
L0 都有 ~100-244× 的嵌入→残差范数跃迁,冻它会因平凡原因摧毁一切;v1 含 L0 的对照数据
`results_freezeL0_*` 证实了这一点)。解读纪律:**只有"冻到 k 仍无损"是干净证据**,深 k 损坏
是分布外冲击的上界;R0(k)=intact−swap 区分"模型坏了"和"图像没用上"。

**结果(`fig_freeze.png`,nc=167/模型):**

| 模型 | freeze_IMG 后果 | freeze_TXT 后果 |
|---|---|---|
| LLaVA-1.5(enc) | acc 微降,**R0 全深度保持基线**(0.10-0.20 vs 0.18) | k=0.09N 即崩(0.19) |
| Qwen2.5-VL(enc) | **0.21N 内 acc 无损**(0.75 vs 0.76),**R0 冻到 0.79N 仍保持**(0.20-0.28 vs 0.28) | 0.2N 崩 |
| Gemma(free) | **0.1N 即崩**(R0 0.38→0.03) | 0.1N 崩 |
| NEO-2B(free) | 0.2-0.3N 崩(R0 0.33→0.04→0.01) | **0.2N 内全场最耐冻**(acc 0.56,R0 保持 +0.20)然后崩 |
| SAIL(free) | 0.2-0.3N 崩(R0 0.31→0.08→0.03) | 0.09N 后崩 |

**双重分离:**
1. **encoder 模型的 LLM 从不需要加工图像 token**——把它们的图像残差流整段冻在投影输出上,
   可用图像信号(R0)在任意深度都完好。encoder 的输出是被"原样读取"的。
2. **native 模型的早层图像加工是因果必要的**——取消它,图像信号在 0.1-0.3N 内归零。
   它们的早层**就是** encoder。
3. **NEO 的 pre-Buffer 分工得到因果确认**:它是唯一能把 text 流冻过 0.2N 而保留大半图像
   信号的模型(其余模型 0.1-0.2N 即崩)——pre-Buffer 里 text 确实接近直通
   (描述性数据:L1-11 的 u_txt=0.17 vs 之后 0.60,L4 仅 0.02)。

**这就把"encoder 有啥用"变成了可测陈述:encoder 的作用 = 让 LLM 的全部深度免于图像编码
工作**(Qwen/LLaVA 冻图无损;NEO 要拿出 30% 的层做 pre-Buffer,其间文本近乎闲置)——
它是计算分工/效率选择,**不改变融合位置**(融合等的是文本查询成熟,见 lastrow),也不
必然改变能力(本基准 Gemma 0.79 全场最高)。

## 3.6 普适分工图(8 native + 4 encoder)与"文本层去哪了"的解答

**`fig_division_of_labor.png`**:每行一个模型(NEO1.0-2B-MT/SFT、NEO1.0-9B-MT/SFT、
NEO1.5-2B/9B、Gemma、SAIL || LLaVA-1.5、LLaVA-NeXT、OneVision、Qwen2.5-VL),
红带=图像流逐层加工强度,紫带=文本流,白虚线=NEO pre-Buffer 边界。
**融合区间 = "融合深度分布"的箱线图**:对 prefix-knockout 曲线取每层边际下降
marg(d)=max(retained(d−1)−retained(d),0)/R0——它是"可用图像信号在哪一层变得不可恢复"
的归一化分布;箱=[q25,q75](中间 50% 的融合发生区),黄点=q50(融合中位深度),
须=[q10,q90](等价于曲线跌破 90%/10% 的稳健版)。该定义无任意阈值、对噪声稳健
(单调 CDF 上取分位),q50 的 bootstrap 95% CI 在大样本配置上紧
(NEO-2B-MT [0.60,0.60]、Qwen [0.54,0.57]、LLaVA [0.22,0.38])。一图可见:除 LLaVA-1.5 外 11/12 个配置的融合框都在中段;native 的红带前段热
(NEO 恰好热到 pre-Buffer 边界),encoder 的红带前段冷;紫带除 NEO 外所有模型前段都热。
**`fig_freeze_bars.png`**:冻结实验的因果摘要(5 模型 ×2 流)。

**关键概念修正("native 前层不处理文本"是误读):Transformer 的层不按模态分工,同一层
对每个位置都计算;分工是按位置流的。**
- 一般 native(Gemma/SAIL)前层**照常加工文本**(Q1 文本工作占比 44%/38%,与 LLaVA 43%/
  Qwen 34% 相当),同时在 image 位置并行做编码——"一层两用",没有文本层缺失。
  因果确认:Gemma/SAIL 冻文本 0.1N 即崩(和 encoder 模型一样)。
- **NEO 是特例且有干净解释:pre-Buffer 是在完整 LLM 前新插入的层,不是挪用的层**
  (NEO-2B = Qwen3-1.7B 的 28 层 + 插入 12 层 = 40;9B = 36+6=42)。文本流水线就是原来那
  28/36 层,整体右移:文本工作 Q1 仅 2.7% → Q2 暴涨 45%。没有文本容量损失,是深度增加。

- knockout 测**已训练电路的依赖**,不是"特征可读性"的上限;早降可能反映浅层通路的脆弱性。
- prefix 封锁期间 image→image 注意力保持完好(图像 token 在封锁中继续成熟),因此
  prefix 单独测的是"读取侧的依赖位置";suffix 与 lastrow 补全了起点与通路归属。
- 全部结论基于 VMCBench MCQ + 字母打分;开放生成/grounding 任务的融合位置可能不同。
- "SAIL 最晚 (0.59)"与 native 带 [0.48,0.62] 部分重叠,应读作"中带偏晚端",非严格序。
- Qwen 因 OOM 跳过的样本集中在文本密集类(DocVQA 等),其 causal 子集与 LLaVA 的构成
  略有偏差(已查:不影响中层结论,类别拆解下逐类皆中层)。
